8. Room Static Data ✅ Fixed
RateHawk Q: Do you show ETG room images & amenities? How do you match rates to room groups?
Second update: We don't have "rg_hash" in static data. We have rg_ext arrays. What is rg_hash?
Our Remark: Thank you for the clarification. We have corrected our matching logic:
  • Yes, we display ETG room images and amenities for ETG rates.
  • Corrected matching process (we no longer use rg_hash):
  1. From static hotel data, we index all room_groups by each group's rg_ext array values — specifically each rg_ext[*].rg value.
  2. From each search rate, we read rg_ext.rg.
  3. We match the rate's rg_ext.rg value against the indexed room_groups[*].rg_ext[*].rg values.
  4. On match → the rate is enriched with the room group's name, description, images, amenities, and bed configuration.
  5. If no match → fallback to the rate's own room_name + placeholder images.
We do NOT use rg_hash — we understand it does not exist in static data. Our code has been updated to use rg_ext.rg for matching exclusively.

25. Errors & Status Processing Logic ✅ Done
RateHawk Q: Fill cross-mapping tables for /finish/ and /finish/status/.
Our Remark:
Table 1: /hotel/order/booking/finish/
ETG API Response Frontend Status (shown to user) Backend Processing Logic
Status "ok" "Processing your booking..." Update booking status to "processing" in database. Immediately begin polling /finish/status/ every 2.5 seconds.
5xx status code "Server error, retrying..." Retry the /finish/ call once after 1 second. If retry also fails, set status to "processing" and proceed to poll /finish/status/ (booking may still succeed on ETG's side).
Error "timeout" "Booking is being processed..." Set booking status to "processing". Proceed to poll /finish/status/. The booking may still complete successfully.
Error "unknown" "Booking is being processed..." Set booking status to "processing". Proceed to poll /finish/status/.
Error "booking_form_expired" "Your session has expired. Please search again." Set booking status to "expired". Show error message and redirect user back to the hotel search page.
Error "rate_not_found" "This room is no longer available. Please select a different room." Set booking status to "failed". Show error and redirect user back to the hotel details page to select a new rate.
Error "return_path_required" "Additional payment verification is required." Display the verification prompt/3DS redirect to the user for additional payment authentication.

Table 2: /hotel/order/booking/finish/status/
ETG API Response Frontend Status (shown to user) Backend Processing Logic
Status "ok" "Booking Confirmed ✅" Save booking status as "confirmed" in Supabase database. Display booking confirmation page using data from prebook step. Send confirmation email with markup prices to customer via Brevo API. Send notification to corporate email.
Status "processing" "Still processing your booking..." (with loading spinner) Continue polling every 2.5 seconds. Do not show any error. Maximum polling duration: 180 seconds.
Error "timeout" "Your booking is being processed. We will email you the confirmation once it is finalized." Save booking status as "pending" (NOT "failed"). Stop polling. The booking may still be confirmed by ETG — send email notification once status is updated.
Error "unknown" "Your booking is being processed. We will email you the confirmation once it is finalized." Save booking status as "pending". Stop polling. Same handling as timeout.
5xx status code "Server error, retrying..." Continue polling (ETG may still be processing the booking). Log the error for internal monitoring. Do NOT mark as failed.
Error "block" "Your payment was blocked by your bank. Please contact your bank or try a different card." Save booking status as "failed". Show user-friendly error message.
Error "charge" "Payment charge failed. Please try again or use a different payment method." Save booking status as "failed". Show user-friendly error message.
Error "3ds" "3D Secure verification failed. Please try again." Save booking status as "failed". Show user-friendly error message.
Error "soldout" "This room was sold out while processing your booking. Please select a different room." Save booking status as "failed". Redirect user to hotel details page.
Error "provider" "The hotel provider encountered an error. Please try again later." Save booking status as "failed". Show user-friendly error and offer retry option.
Error "book_limit" "Booking limit reached for this property. Please try a different hotel." Save booking status as "failed". Show user-friendly error.
Error "not_allowed" "This booking is not permitted at this time. Please contact support." Save booking status as "failed". Show user-friendly error with support contact.
Error "booking_finish_did_not_succeed" "Booking could not be completed. Please try again or select a different room." Save booking status as "failed". Show user-friendly error and offer retry option.
